Good Times with CSS Positioning

How to Avoid Hurting Your Computer

February 14, 2015

Over the past week as the Phase 0 curriculum has introduced us to HTML and CSS, I have admittedly been struggling with CSS layouts and, specifically, the position property and its values. There was a moment, thankfully a brief moment, during one of this week's challenges where throwing my computer out the window was looking like a logical solution. The exercise was to work through various repositionings of various div/ box elements. I finished the challenge with the aid of TA office hours and some of my cohorts' guidance (props to the awesomeness of the DBC support system!). And now that this week's technical blog post is to address an aspect of this week's challenges, I'm tackling CSS positioning. So, here goes my explanation (with the help of a few awesome resources!).

First thing to know about the CSS position property is its function. The position property allows you to tell a particular HTML element how you want it to work within the flow of the other elements and their properties. In essense it allows you to arrange your various HTML elements in all sorts of ways! It allows you to break out of the vertical, stacked-on-top-of-each-other default.

Four primary values can be used for the position property; they include: static, relative, absolute, and fixed. Static is the default value of all elements. Using a static value for the position of your element leaves you in the default mode of everything being linear and vertical. When your element is using the static position, you are unable to utlize the offset properties of left, right, top, and bottom. These offset properties are how you tell your element where and how to position itself within the DOM, or generally your Internet browser's window. You cannot use these offset properties until you give your element a non-static position value.

So! First non-static position value to discuss: relative! When you assign an element the position value of relative, you give it the ability to use the offset properties of left, right, top, and bottom, and, therefore, the ability to move it about the page. Important to note about the relative position value is that it also grant an element the ability to use a coordinate system other than the default document coordinate system. Basically, what this means is that you can take and element with a relative position and give it different point of reference to how it positions itself. This is done by placing the relative-positioned element within another element. So, an element with a relative position with an offset property of top: 400px will use the coordinate system of its parent element.If its parent element is the document its top will be 400px from the top of the document. If its parent element is, say, a content element that is already positioned 400px from the top of the document, it will be 400px from the content element, instead of 400px from the document.

Moving on to the next position value...absolute. An element that is given the position value of absolute is taken out of the normal flow of the document. I love the simile href="http://alistapart.com/article/css-positioning-101">Noah Stokes uses: "If the relative value acts like Superman, then the absolute value mirrors Inception—a place where you design your own world." The absolute position value gives an element the ability to be placed anywhere within the document using the offset properties without disrupting the placement of any other elements or the document's flow. Furthermore, the absolute position value creates a new coordinate system for any child elements it has, therefore taking those child elements also out of the document's normal flow (This aspect of the absolute value is where I was running into problems during my challenge and had to fight off the computer-tossing urge. Hooray! for having this problem and now being able to articulate why! ::cabbage patch dance::).

Last, but indeed, certainly not least, is the position value of fixed. A fixed position value, like its name suggests, fixes an element with offset properties as they relate to the window. This last part -- about "as they relate to the window"-- is important to distinguish for this is as opposed to the element relating to its parent element. Therefore, a fixed position value gives an element the ability to stay-put even while you scroll through a window.

Okay, I kind of fibbed...there are two more values for the position property-- inherit and initial. Inherit assigns there element the position value of its parent element -- pretty self explanatory here. Nonetheless, this is important to note, because, as was noted at the beginning, an element's default position value is static regardless its parent's position value. And, lastly, for real this time, the initial value reverts the position value back to its default.

Phew! I think I did it! I just explained the various values of the CSS position property. There's always of course room for improvement (which goes to say I welcome your feedback and clarification!). But at least for now, the urge to throw my computer out the window on the account of CSS positioning has subsided...
