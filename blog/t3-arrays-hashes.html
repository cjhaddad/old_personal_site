Arrays and Hashes

Two of the Coolest Ruby Objects, IMHO

February 20, 2015

Arrays and hashes are two types of objects in the Ruby programming languange. Yet, unlike individual objects like an integer or a string, they are container objects. This means that arrays and hashes act as collections of objects, where you can assign them a whole heap of information in the form of any object (including arrays and hashes!). First, I'll describe more thoroughly the array object and then the hash object.

An array in the simplest of descriptions can be thought of as a list of objects. Here's an example of a variable named animals with an array of strings assigned to it: 

            
              animals = ["monkey", "kangaroo", "octopus"]
Visually, we can regonize an array object through its use of square brackets: [] . Inside of the array above are three string objects. Arrays use an ordered index system to identify each of the objects within it. Ordered means that each object in an array is given an index, which in the case of an array is a number, that follows normal seqeuntial counting. HOWEVER! By "normal" I'm referring to the programming norm of counting that always begins at 0. Therefore, using the animal array example above, the "monkey" string object would have an index of 0; the "kangaroo" string object would have an index of 1; and the "octopus" string object would have an index of 2. Subsequent objects after "octopus" would continue the sequential numerical indexing.

Hashes like arrays use an ordered, index system to identify the information within it. Hashes begin to differ from arrays when it comes to how it stores objects. Hashes store information by a key/ value pair. Here's an example of a variable named my_pet with a hash assigned to it: 

            
              my_pet = { "name" => "Rufus",
                         "type" => "dog",
                         "breed" => "chihuahua",
                         "favorite food" => "ALL THE MEATS" }
In hash variable my_pet "name", "type", "breed", and "favorite food" are keys, where as "Rufus", "dog", "chihuahua", and "ALL THE MEATS" are the keys' respective values. A hash key works as an index of a hash value. The key and value of a hash pair can be any kind of object -- in this example I used strings. This is important to note, for it is here where a major difference between hashes and arrays is occurs. A hash key is like an array index, but instead of only being an index identified by an integer it can be any object (including an integer). Another interesting aspect of a hash's index system is that there is also a meta-index, where each key/ value pair has a numerical index, just like the objects in an array. Similar to arrays the meta-index system of hashes follows a sequential numbering. So, the key/value pair of "name" => "Rufus" has a meta-index of 0, and so on.

Arrays and hashes provide powerful ways to store and access data. Depending on the context and desired uses of various information, both arrays and hashes offer dynamic modes of working with data in our programs.
