Brief Intro to Regex

March 27, 2015

Regular expressions are a super helpful tool to identify and search for a specific pattern within a string. Ruby, in addition to Perl and JavaScript, has built-in capabilities to use regular expression -- this means you do not need to use a standard library to utilize regular expressions(this is the case for C++, Python, and Java.). In Ruby you can create a regular expression -- regexp or regex -- by the /.../ and %r{...} literals or by using the Regexp.new constructor. You can use regular expressions to search for patterns to replace that pattern with another set of text or extract the pattern and save it to use for something else. There are tons of different scenarios where the use of regular expressions can help you.

When you create a regex, you use a mix between meta characters and regular characters to identify the string pattern for which you are searching. Meta-characters include: (,),[,],{,},.,?,+,*. A variety of methods are built into the Ruby Regexp Class and, therefore, can be called on your regex. The =~ operator works the same way as the #match does by looking for a match between a regex expression and a string. The return value when using the =~ operator is either nil, if no match is found, or the index of the first matching character. When using #match, a Match Data object is returned. 

          
            >> my_regex = /pat/
            >> my_string = "Does this have a pattern?"

            >> my_regex =~ my_string
              17

          >> my_regex.match(my_string)
            #

Character classes can be inserted into a regular expression to allow you to find each of those characters separately or within a range, i.e. [agw] and [a-d]. Character classes are denoted by the square brackets []. I'll use #scan to illustrate how to extract characters from a string using regex. 

          
            >> p "hello, is it me you're looking for?".scan(/[esr]/)
              ["e", "s", "e", "r", "e", "r"]
